## 页面路由结构

### 1. 登录页面
- **路径**: `/login`
- **组件**: `Login.vue`
- **功能**: 用户身份验证
- **特点**: 无需认证即可访问

### 2. 主布局页面
- **路径**: `/layout`
- **组件**: `Layout.vue`
- **功能**: 提供主界面框架
- **子路由容器**: 所有受保护页面的父容器

#### 2.1 仪表盘页面
- **路径**: `/layout/dashboard`
- **组件**: `Dashboard.vue`
- **功能**: 显示系统概览信息
- **权限**: 已认证用户

#### 2.2 账户管理页面
- **路径**: `/layout/accounts`
- **组件**: `accounts/Accounts.vue`
- **功能**: 管理系统账户
- **权限**: 管理员

#### 2.3 用户管理页面
- **路径**: `/layout/users`
- **组件**: `users/Users.vue`
- **功能**: 管理系统用户信息
- **权限**: 已认证用户

#### 2.4 组织管理页面
- **路径**: `/layout/orgs`
- **组件**: `orgs/Orgs.vue`
- **功能**: 管理组织架构
- **权限**: 已认证用户

#### 2.5 学员管理页面
- **路径**: `/layout/students`
- **组件**: `students/Students.vue`
- **功能**: 管理学员信息
- **权限**: 已认证用户

## 状态管理结构

### Auth Store (stores/auth.js)
- `user`: 当前用户信息
- `accessToken`: 访问令牌
- `refreshToken`: 刷新令牌
- `isAuthenticated`: 认证状态
- `authChecked`: 认证检查状态

**Actions**:
- `initializeAuth()`: 初始化认证状态
- `setTokens()`: 设置令牌
- `setUser()`: 设置用户信息
- `logout()`: 登出操作
- `checkAuthStatus()`: 检查认证状态

## API 接口结构

### 认证相关
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/logout` - 用户登出
- `GET /api/auth/refresh-token` - 刷新令牌

### 用户相关
- `POST /api/user/self/` - 获取当前用户信息
- `POST /api/user/list/` - 获取用户列表
- `POST /api/user/add/` - 添加用户
- `POST /api/user/edit/:id` - 编辑用户
- `POST /api/user/detail/:id` - 获取用户详情

### 其他管理接口
- `POST /api/account/` - 账户相关
- `POST /api/org/` - 组织相关
- `POST /api/student/` - 学员相关

## 权限控制

### 路由权限
- `requiresAuth`: 需要认证才能访问
- `requiresGuest`: 游客权限（如登录页面）

### 组件权限
- 根据用户角色显示/隐藏特定功能
- 操作权限验证

## 响应式设计

### 断点
- 移动端: < 768px
- 平板端: 768px - 1024px
- 桌面端: > 1024px

### 适配策略
- 侧边栏在移动端折叠
- 表格在小屏幕下可横向滚动
- 按钮和表单项自适应尺寸
